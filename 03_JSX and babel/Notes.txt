//                                      SUMMARY NOTE - JSX and Babel

// NOTE-1: Babel is a JavaScript compiler/transpiler that converts modern JavaScript (ES6+) and JSX syntax
//         into backward-compatible JavaScript code that browsers can understand and execute.

// NOTE-2: For using Babel in development, include the Babel CDN in the <head> tag and set type="text/babel"
//         on your custom JS file script.
//
//         Normally, the default value of the type attribute is "text/javascript", so the browser:
//         1. Sends request for the JS file
//         2. Downloads it
//         3. Executes it directly as JavaScript
//
//         But when we set type="text/babel" (or any non-JavaScript MIME type), the browser still sends
//         the request and downloads the file, but it does NOT execute it directly as JavaScript.
//
//         Instead, Babel reads that file, transforms the JSX/modern JS syntax into browser-compatible
//         JavaScript (like React.createElement() syntax), and then executes the transformed code.

// NOTE-3: So technically, the browser itself still sends the request for our JS file — Babel does not.
//          After the browser downloads the file, Babel intercepts the script having type="text/babel",
//          reads its content, transforms it into browser-compatible JavaScript, and then executes it.
//
//          Babel also generates source maps. These source maps help DevTools show our original source code
//          (JSX/modern JS) instead of the transformed JavaScript code while debugging.
//
//          This behavior depends on the "Enable JavaScript source maps" setting in browser DevTools.

// NOTE-4: When Babel processes a JS file containing JSX syntax, it first receives the entire file content
//          internally as plain text/string format, then parses and transforms it into browser-compatible JavaScript.

// NOTE-5: To use JavaScript expressions or variables inside JSX, we put them inside {} curly braces.
//          Curly braces mean: "switch from JSX mode into JavaScript mode".
//
//          Example:
//          <h1>{userName}</h1>
//
//          Babel transforms JSX into React.createElement() syntax, where those JS expressions are evaluated
//          and inserted as values in the transformed code.
//
//          Inside {} we can write JavaScript expressions like:
//          {name}
//          {2 + 3}
//          {isLoggedIn ? "Yes" : "No"}
//
//          But not statements like:
//          if(){}
//          for(){}
//
//          because JSX expects a value/expression inside {}.

// NOTE-6: If we put console.log() inside {} curly braces, it still runs because it's valid JavaScript code.
//          But console.log() returns undefined, and React does not render values like:
//
//          undefined
//          null
//          true
//          false
//
//          on the browser UI.
//
//          Example:
//          <h1>{console.log("Hello")}</h1>
//
//          Here:
//          1. "Hello" prints in console
//          2. console.log() returns undefined
//          3. React ignores undefined during rendering, so nothing appears on screen
//
//          If we want to display them visibly, we can:
//          1. Write them directly as text
//          2. Or convert them into strings inside {}
//
//          Example:
//          <h1>{"false"}</h1>
